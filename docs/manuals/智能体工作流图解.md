## 3. 子流程: 规则与参数配置 (Micro: Rules & Parameters)

此图展示了在工作流中流转的数据（参数）和保障质量的规则（Guardrails）是如何生效的。

```mermaid
sequenceDiagram
    participant Input as 输入源 (User/API)
    participant Workflow as 工作流引擎
    participant Agent as 智能体
    participant Guard as 校验器 (Guardrails)
    
    note right of Input: 全局参数 (Global Params)<br>e.g. { "topic": "AI", "depth": "deep" }
    
    Input->>Workflow: 启动工作流 (Start)
    Workflow->>Workflow: 解析全局参数 {{params}}
    
    Workflow->>Agent: 节点执行 (Node Execution)
    note right of Workflow: 注入上下文变量 (Variables)<br>e.g. {{input.topic}}
    
    Agent->>Agent: LLM 推理
    Agent-->>Guard: 输出原始结果 (Raw Response)
    
    Guard->>Guard: 1. 结构校验 (JSON Schema)
    alt 结构错误
        Guard-->>Agent: 触发重试 (Retry)
    end
    
    Guard->>Guard: 2. 内容校验 (Content Rules)<br>(敏感词/置信度/必填项)
    
    alt 校验通过
        Guard-->>Workflow: 返回标准化结果
    else 校验失败
        Guard-->>Workflow: 标记节点 Failed
    end
```

## 4. 核心流程: 工作流编排与执行 (Workflow Orchestration & Execution)

此图展示了多个智能体如何串联协作，完成复杂任务。

```mermaid
stateDiagram-v2
    [*] --> Init: 接收触发信号
    
    state "初始化阶段" as Init {
        LoadParams: 加载全局参数
        ValidateInput: 校验输入数据
    }
    
    Init --> Analyze: 进入分析节点
    
    state "节点A: 市场分析 (Agent)" as Analyze {
        Retrieval: 检索知识库
        Reasoning: LLM 推理
        OutputA: 输出分析报告
    }
    
    Analyze --> Decision: 条件分支
    
    state Decision <<choice>>
    Decision --> DeepDive: 风险高
    Decision --> Summary: 风险低
    
    state "节点B: 深度研判 (Debate)" as DeepDive {
        Round1: 多方辩论
        Round2: 观点碰撞
        Judge: 裁判总结
    }
    
    DeepDive --> Summary
    
    state "节点C: 生成摘要 (Agent)" as Summary {
        Format: 格式化输出
    }
    
    Summary --> [*]: 流程结束
```
