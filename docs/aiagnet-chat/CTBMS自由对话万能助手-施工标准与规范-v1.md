# CTBMS 自由对话万能助手施工标准与规范 v1.0

## 1. 文档目的

本规范用于统一“自由对话万能助手”改造的研发施工方式，确保：

1. 需求实现一致。
2. 技术质量可控。
3. 上线风险可控。
4. 可持续迭代。

---

## 2. 施工总原则

1. **体验优先**：普通用户无感，专业能力后台化。
2. **复用优先**：先查已有能力，再做临时生成。
3. **契约优先**：所有输入输出必须有 schema。
4. **最小暴露**：普通用户不可见内部路由和治理细节。
5. **可回滚**：每项改造都必须具备降级路径。

---

## 3. 架构施工规范

## 3.1 分层边界

1. Web 层：仅处理交互和展示，不写业务决策。
2. Conversation 层：负责会话编排、状态推进、回复封装。
3. Capability 层：负责能力检索、复用、临时生成、治理。
4. Runtime 层：执行 workflow/skill，不让 LLM 直接越层执行。

## 3.2 依赖规范

1. 类型定义统一来自 `@packages/types`。
2. 新增接口必须补类型，前后端同步升级。
3. 业务逻辑禁止绕过服务层直接写控制器。

---

## 4. 前端施工规范

## 4.1 对话输入与回显

1. 发送后必须立即本地回显。
2. 输入框立即清空。
3. 必须提供处理中占位与失败提示。
4. 失败后必须给“重试发送”动作。

## 4.2 回复卡片规范

1. 助手回复必须支持 `replyOptions`。
2. `replyOptions` 最多 4 个，最少 1 个。
3. 按钮文案必须是中文口语短句。
4. 普通用户不展示内部状态码与 ID。

## 4.3 文案规范

1. 禁止面向普通用户使用术语：plan、draft、grant、route、governance。
2. 错误提示必须给“可执行下一步”。
3. 所有英文状态在普通区必须中文化。

---

## 5. 后端施工规范

## 5.1 createTurn 响应规范

必须返回：

1. `assistantMessage`
2. `state`
3. `replyOptions`（必备）

可选返回：

1. `missingSlots`
2. `autoExecuted`
3. `executionId`

## 5.2 replyOptions 规范

1. `mode` 仅允许 `SEND` / `OPEN_TAB`。
2. `OPEN_TAB.tab` 仅允许 `progress/result/delivery/schedule`。
3. 缺信息场景必须含“默认值继续”类选项。

## 5.3 LLM 增强规范

1. LLM 输出必须 JSON 化并做解析兜底。
2. LLM 不可用时必须自动回退规则选项。
3. 增强功能必须有开关（环境变量可关闭）。

---

## 6. 能力复用与临时生成规范

## 6.1 复用流程（强制）

1. Owner Pool 检索
2. Team/Public Pool 检索
3. 命中则复用，不重复创建

## 6.2 临时能力规范

1. 临时能力必须带 TTL。
2. 临时能力必须记录来源问题指纹。
3. 临时能力必须可自动清理。

## 6.3 晋升规范

满足以下条件可晋升正式能力：

1. 复用率达标
2. 成功率达标
3. 风险等级合规

---

## 7. 测试施工规范

## 7.1 必测项

1. 发送即回显（前端交互）
2. `replyOptions` 契约（API）
3. 自由对话主链路（e2e）
4. 复用优先与临时生成路径（e2e）

## 7.2 门禁项

1. `pnpm --filter api run type-check`
2. `pnpm --filter web run type-check`
3. `pnpm --filter api run test:e2e:agent-suite`
4. `pnpm agent:pre-release`

---

## 8. 可观测施工规范

1. 路由决策必须可追踪（来源、分数、原因）。
2. 临时能力必须可追踪（创建时间、TTL、命中率）。
3. 用户体验指标必须可追踪（首结果时长、追问次数、发送转化）。

---

## 9. 发布与回滚规范

## 9.1 灰度发布

1. 10% -> 30% -> 60% -> 100%
2. 每阶段观察至少 24-48 小时

## 9.2 回滚策略

1. 关闭 LLM 增强，回退规则选项。
2. 关闭乐观回显，回退服务端回包渲染。
3. 保留后台复用与治理，不回滚数据结构。

---

## 10. DoD（Definition of Done）

每个施工项必须满足：

1. 功能完成并通过 type-check。
2. 对应测试补齐并通过。
3. 用户文案无专业术语泄露。
4. 有监控指标与回滚开关。
5. 文档同步更新。
