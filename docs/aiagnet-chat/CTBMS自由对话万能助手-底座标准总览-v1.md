# CTBMS 自由对话万能助手底座标准总览 v1.0

## 1. 目标

本标准用于把“对话助手”升级为“可生长的万能助手底座”，确保：

1. 自由对话可稳定落地到系统能力执行。
2. 能力复用优先，临时生成可控。
3. 普通用户无感，平台可治理可审计。

---

## 2. 标准体系结构

底座标准由 7 个子标准组成：

1. 工作流标准：`docs/aiagnet-chat/CTBMS自由对话万能助手-底座标准-工作流-v1.md`
2. Skills 标准：`docs/aiagnet-chat/CTBMS自由对话万能助手-底座标准-Skills-v1.md`
3. 参数与规则标准：`docs/aiagnet-chat/CTBMS自由对话万能助手-底座标准-参数与规则-v1.md`
4. 数据源标准：`docs/aiagnet-chat/CTBMS自由对话万能助手-底座标准-数据源-v1.md`
5. 风控标准：`docs/aiagnet-chat/CTBMS自由对话万能助手-底座标准-风控-v1.md`
6. 输出与交付标准：`docs/aiagnet-chat/CTBMS自由对话万能助手-底座标准-输出与交付-v1.md`
7. 质量与治理标准：`docs/aiagnet-chat/CTBMS自由对话万能助手-底座标准-质量与治理-v1.md`

---

## 3. 统一施工要求（跨标准）

1. 所有能力必须有明确输入输出 Schema。
2. 所有能力必须标注风险等级与可见范围（private/team/public/ephemeral）。
3. 所有能力执行必须可观测（成功率、时延、成本、错误码）。
4. 所有临时能力必须具备 TTL 与回收机制。
5. 所有用户可见输出必须“人话化”，禁止内部术语直出。

P1（第二阶段）

2. 数据源标准
3. 质量与治理标准

---

## 5. DoD（标准落地完成定义）

1. 标准文档可执行（非概念性描述）。
2. 每个标准都有字段清单、校验规则、错误处理策略。
3. 对应 API/类型定义已补齐或有明确改造任务。
4. 有最小验收脚本与门禁命令。
