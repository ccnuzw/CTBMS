# CTBMS 自由对话万能助手底座标准-质量与治理 v1.0

## 1. 质量指标体系

### 1.1 体验指标

1. 首次结果时长（P95）
2. 用户追问次数
3. 会话完成率
4. 发送转化率

### 1.2 能力指标

1. 复用命中率
2. 临时能力创建率
3. 临时能力晋升率
4. 临时能力淘汰率

### 1.3 运行指标

1. 执行成功率
2. 平均时延
3. 平均成本
4. 回退触发率

## 2. 治理规则

1. 同类能力指纹去重，禁止重复膨胀。
2. 低质量临时能力自动淘汰。
3. 高复用高质量能力自动提案晋升。
4. 高风险能力必须经过审批闭环。

## 3. 审计要求

1. 路由决策可追踪。
2. 执行过程可回放。
3. 参数规则变更可溯源。

## 4. 发布门禁

1. `pnpm --filter api run type-check`
2. `pnpm --filter web run type-check`
3. `pnpm --filter api run test:e2e:agent-suite`
4. `pnpm agent:pre-release`

## 5. 验收标准

1. 关键指标可视化。
2. 治理动作可自动化执行。
3. 灰度与回滚演练可复现。

## 6. 晋升批次治理补充规范

1. 批次失败项必须支持按错误码聚合、筛选、导出。
2. 失败重放默认采用 `RETRYABLE_ONLY`，仅重放可重试错误。
3. 可重试策略由白名单控制，不可重试策略由黑名单控制，均可在管理端配置。
4. 推荐提供两套预设：`网络波动型` 与 `严格型`，避免运维手工维护错误码。

## 7. 快速回归基线（迭代期）

1. 常规迭代默认执行：`pnpm --filter web run type-check`。
2. 若 API 代码有改动，至少执行：`pnpm --filter api run type-check`。
3. 若存在仓库内无关编译阻塞，可先完成本次功能验证并在发布前集中修复阻塞项。
4. 全量 e2e 与 `pre-release` 保留到里程碑节点执行。
