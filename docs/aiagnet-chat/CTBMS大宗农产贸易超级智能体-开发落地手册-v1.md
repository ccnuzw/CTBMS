# CTBMS 大宗农产贸易超级智能体开发落地手册 v1.0

- 文档类型：实施手册（Execution Playbook）
- 目标：将 PRD/TDD/API/DDL 转化为可执行开发流程
- 适用周期：M1-M4

## 1. 实施总原则

1. 复用优先：先 `REUSE`，再 `EXTEND`，最后 `NEW`。
2. 单写入统一读取：采集侧不扩散写入，分析侧统一标准层读取。
3. 双跑先行：任何切流都先对账，达标再切。
4. 可回滚：每次变更必须可一键回退到稳定链路。

## 2. M1 落地步骤（4 周）

### Week 1：盘点与冻结

1. 盘点存量资产：`PriceData`、`FuturesQuoteSnapshot`、`MarketIntel`。
2. 冻结标准映射版本 `mapping_v1`。
3. 冻结首批核心指标与口径版本 `metric_v1`。
4. 明确切流阈值（误差、缺失、冲突、成功率）。

交付物：

- 映射清单（字段级）
- 指标口径清单（版本化）
- 门禁阈值配置

### Week 2：适配层与标准读取

1. 实现标准化适配层（Price/Futures/Intel）。
2. 为工作流增加标准读取开关（按模板灰度）。
3. 对话结果增加标准化数据来源标识。

交付物：

- 标准读取服务 API
- 灰度开关配置
- 第一版联调报告

### Week 3：双跑与对账

1. 启动双跑任务（legacy vs standard）。
2. 输出对账报告（日级）：误差率、缺失率、冲突率。
3. 补齐异常样本处理和映射规则修订。

交付物：

- 对账任务结果存档
- 异常处理清单
- `mapping_v1.1`（如有）

### Week 4：试点切流与回滚演练

1. 选 1 条模板（建议周报）进行试点切流。
2. 完成切流前后质量对比。
3. 完成回滚演练并记录。

交付物：

- 切流审批记录
- 回滚演练报告
- M1 验收结论

## 3. M2-M4 扩展路径

### M2（对话与模板）

1. 完成证据包固化输出。
2. 完成周报/预警模板标准层读取。
3. 对话执行链路接入门禁结果提示。

### M3（天气物流）

1. 接入天气/物流并纳入标准层。
2. 增加天气扰动与运输摩擦指标。
3. 完成多源冲突仲裁策略。

### M4（运营闭环）

1. 完成订阅、回测、投递联动。
2. 完成质量看板和审计看板。
3. 完成灰度全量与 legacy 收敛计划。

## 4. 开发节奏与会议机制

1. 日会：同步阻塞项和门禁指标。
2. 周会：对账报告评审 + 切流决策。
3. 里程碑评审：M1/M2/M3/M4 出入口门禁检查。

## 5. 变更控制（Change Control）

1. 映射规则变更必须升级 `mapping_version`。
2. 指标口径变更必须升级 `metric_version`。
3. 任一版本升级必须附带影响评估与回滚方案。

## 6. 发布门禁清单

1. 功能门禁：核心 Story 验收通过。
2. 质量门禁：关键质量指标达标。
3. 对账门禁：双跑阈值连续达标。
4. 稳定性门禁：错误率与时延无显著回归。
5. 回滚门禁：回滚演练成功并可复现。

## 7. 推荐执行命令（团队统一）

1. `pnpm lint`
2. `pnpm type-check`
3. `pnpm --filter api test`
4. `pnpm --filter api test:e2e`

## 8. 关键风险与应对

| 风险         | 触发信号             | 应对                      |
| ------------ | -------------------- | ------------------------- |
| 双口径并存   | 同指标多版本冲突上升 | 强制标准层读取 + 对账门禁 |
| 映射规则漂移 | 对账失败连续上升     | 冻结版本 + 回退稳定映射   |
| 性能回归     | 标准层查询时延超阈值 | 读缓存 + 索引优化 + 降级  |
| 发布风险     | 灰度异常率升高       | 自动回滚 + 事件复盘       |
