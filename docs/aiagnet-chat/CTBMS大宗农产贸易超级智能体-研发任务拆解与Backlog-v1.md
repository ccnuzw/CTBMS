## 0. 文档治理与执行门禁（2026-03-06）

### 0.0 对齐文档

1. 文档治理与执行计划：`docs/aiagnet-chat/CTBMS对话产品文档治理与落地执行计划-v1.md`
2. 功能菜单路由映射：`docs/aiagnet-chat/CTBMS对话能力-功能菜单路由映射表-v1.md`

### 0.1 不可变需求（硬约束）

1. 用户主入口固定为 `智能体工作流 > 对话助手`（`/workflow/copilot`）。
2. 普通用户不学习流程配置，使用自然语言直接完成任务。
3. 每轮助手回复必须包含“结论 + 依据 + 下一步动作”。
4. 对话内可完成“提问 -> 得结果 -> 发送/定时”闭环。

### 0.2 Story 入 Sprint 前新增检查项

1. 必须标注 `entryImpact`：
   - `USER_MAIN`：影响普通用户主链路（`/workflow/copilot`）
   - `ADMIN_ONLY`：仅影响管理入口（如 `/workflow/advanced`、`/system/config/*`）
   - `NO_UI`：纯后端或数据能力
2. 标注 `USER_MAIN` 的 Story 不允许新增平行用户入口或绕开对话主链路。
3. 标注 `USER_MAIN` 的 Story 验收必须包含“下一步动作可执行”检查。
4. 不满足以上条目，禁止进入开发。

## 0. 使用说明

1. 本文档用于产品、研发、测试、运营共同对齐范围和节奏。
2. Story 优先级采用 `P0/P1/P2`。
3. 估算单位采用 `SP`（Story Points），用于相对复杂度评估。
4. 每个 Story 至少包含一个可验收输出，不允许“仅代码无产出定义”。

### 0.1 执行标签规范（用于防重复建设）

| 标签     | 含义                         | 使用场景                               |
| -------- | ---------------------------- | -------------------------------------- |
| `REUSE`  | 复用既有模块，不新增平行能力 | 会话/订阅/回测/导出等已有链路          |
| `EXTEND` | 在现有模块扩展能力           | DataConnector、Workflow 节点、规则门禁 |
| `NEW`    | 仅在无现成能力时新增         | 天气/物流事实域、对账任务实体          |

执行原则：

1. 所有 P0 Story 默认先评估 `REUSE`，再评估 `EXTEND`，最后才允许 `NEW`。
2. 新增 Story 进入 Sprint 前必须标注执行标签和影响模块。
3. 无执行标签的 Story 不允许进入开发。

## 5. 验收与 DoD

## 5.0 Story Ready 定义（DoR）

1. 主线四问检查已完成，且不偏离用户主链路目标。
2. 需求验收标准、执行标签（`REUSE/EXTEND/NEW`）已明确。
3. `entryImpact` 已标注（`USER_MAIN/ADMIN_ONLY/NO_UI`）。
4. 影响模块、数据契约、错误码与权限范围已定义。
5. 测试策略已选择（单测/集成/E2E）并给出最小验收样例。
6. 回滚路径已定义（开关、配置、脚本或步骤）。
7. 对外接口变更已评估兼容性。

## 5.1 Story Done 定义

1. 产品验收标准全部满足，且有可演示结果。
2. API 与类型定义同步更新（含 `packages/types`）。
3. 至少一条自动化测试覆盖关键路径。
4. 监控与日志字段按规范落地。
5. 文档更新（PRD/TDD/API/DDL 或操作手册）。
6. 若 `entryImpact=USER_MAIN`，必须验证“下一步动作可执行”且不新增平行入口。

## 5.2 MVP Exit Gate

1. P0 Story 全部完成。
2. 关键质量指标达到 PRD 目标阈值。
3. 至少完成 2 个业务团队真实试运行。
4. 上线门禁、回滚预案、告警链路全部验证通过。
5. 主线四问在最终发布评审中全部为“是”。

## 5.3 切流与下线清单（数据底座）

1. 双跑对账连续 7 天达标（误差、缺失、冲突在阈值内）。
2. 标准层读取覆盖率达到目标阈值（核心模板 >= 90%）。
3. 切流前完成回滚演练并通过。
4. legacy 直读路径下线采用“先禁新用、后读保留、再归档”三步走。

## 6. 风险与缓解

| 风险             | 触发条件                    | 缓解措施                            |
| ---------------- | --------------------------- | ----------------------------------- |
| 外部 API 不稳定  | 连续 3 次超时/错误率 > 阈值 | 缓存回退 + 连接器熔断 + 备用源      |
| 范围膨胀         | 新需求持续插入 P0           | 严格按 Story 优先级和里程碑冻结范围 |
| 指标口径争议     | 同指标跨团队定义不同        | 指标字典评审机制 + 版本化口径       |
| 结果可信争议     | 引用不足或冲突未处理        | 引用门禁 + 冲突说明强制输出         |
| 平行能力重复建设 | 新增独立 API/模型与现有重叠 | 复用现有会话域，新增仅进入标准层    |
