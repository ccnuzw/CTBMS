# CTBMS 自由对话万能助手底座标准-输出与交付 v1.0

## 1. 输出分层标准

1. 用户层输出：结论、依据、下一步（人话）。
2. 系统层输出：执行细节、路由日志、审计信息（后台可见）。

## 2. 用户层输出规范

1. 先结论，后依据，再动作建议。
2. 禁止输出内部术语（plan/draft/route/grant 等）。
3. 错误提示必须提供可执行下一步。

## 3. replyOptions 规范

1. 每轮助手回复至少 1 个选项，最多 4 个。
2. 文案必须短句、动作导向。
3. 选项类型仅允许 `SEND` / `OPEN_TAB`。

## 4. 交付渠道标准

1. 渠道统一：EMAIL/DINGTALK/WECOM/FEISHU。
2. 优先读取默认配置；缺失时给补充引导。
3. 支持原文件与导出文件两种模式。

## 5. 验收标准

1. 普通用户 60 秒内可完成“提问 -> 结果 -> 发送”。
2. 所有渠道失败均有可操作提示。
3. 渠道交付记录可审计。
