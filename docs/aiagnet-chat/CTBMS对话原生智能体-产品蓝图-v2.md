# CTBMS 对话原生智能体产品蓝图 v2.0

> 修订说明（2026-02-26）：针对“普通用户无感 + 自由式对话 + LLM 主驾驶”目标，已新增专项文档：`docs/aiagnet-chat/CTBMS自由对话万能助手-产品设计-v1.md`、`docs/aiagnet-chat/CTBMS自由对话万能助手-施工标准与规范-v1.md`、`docs/aiagnet-chat/CTBMS自由对话万能助手-施工任务分解-WBS-v1.md`、`docs/aiagnet-chat/CTBMS自由对话智能助手-改造升级落地计划-v1.md`。本文档继续作为总体蓝图，详细施工以专项文档为准。

## 1. 目标重定义

- 从“对话触发工作流”升级为“对话即系统本身（Chat-Native）”。
- 用户只需自然表达目标，系统自动补槽、执行、复用中间结果、交付与订阅。
- 对齐已确认策略：
  1. 默认模式：全自动执行。
  2. 能力生长：低风险先用后审。
  3. 交付通道：邮件 + 钉钉 + 企业微信 + 飞书全部支持。

## 2. 产品核心原则

1. 单一入口：所有能力均由对话触发，不要求用户理解节点与配置。
2. 结果可复用：任一轮次产物都可被后续轮次引用、改写、再交付。
3. 先完成任务：默认自动推进，仅在高风险动作弹出确认。
4. 能力可进化：对话中发现缺口，自动提案技能并快速落地。
5. 可审计可治理：全链路留痕，可回放、可降级、可回滚。

## 3. 目标能力（对应用户 5 条诉求）

### 3.1 自然对话直达结果

- 输入一句话后自动完成：意图识别 -> 槽位补齐 -> 执行 -> 结果解释 -> 下一步建议。
- 不再要求用户显式点击“计划确认”等流程按钮（转为系统内部确认点）。

### 3.2 智能体自生成能力与 Skill 能力

- 对话中识别 capability gap 并自动生成 Skill Draft。
- 低风险 Skill 可在当前会话“先使用”并标记为 `PROVISIONAL`。
- 审批通过后转正式 Skill；未通过自动回退并提示替代方案。

### 3.3 无配置多轮 + 中间结果复用

- 引入“会话资产图谱”概念（Conversation Asset Graph）。
- 每轮产物沉淀为可引用对象：`结论/事实表/图表/导出文件/回测结果/冲突摘要`。
- 用户可自然引用：
  - “用刚才第二版结论继续回测”
  - “把上次那份原始 JSON 发给飞书群”

### 3.4 多形态交付

- 聊天框内直接展示是默认交付。
- 文件化交付支持：PDF/Word/JSON/原文件。
- 渠道化交付支持：Email、DingTalk、WeCom、Feishu（统一适配器）。

### 3.5 对话式定时任务

- 支持自然语言创建/修改/暂停/恢复任务。
- 示例：“每周一 9 点把上周玉米复盘发到企微群，并附原始数据文件”。

## 4. 系统架构（对话原生）

1. 对话内核层（Conversation Kernel）
   - 意图解析、槽位管理、澄清策略、会话记忆、风险判断。
2. 能力编排层（Capability Router）
   - Tool/Skill 路由、低风险即时启用、策略门禁。
3. 会话资产层（Asset Graph）
   - 中间结果版本化、引用关系、可追溯血缘。
4. 交付与自动化层（Delivery + Scheduler）
   - 多渠道发送、定时任务、失败重试、审计记录。

## 5. 风险分级与“先用后审”规则

### 5.1 风险级别

- L1 低风险：只读查询、聚合分析、格式转换、报告生成。
- L2 中风险：外发消息、定时调度、跨组织分享。
- L3 高风险：写库、交易、调用有副作用外部系统。

### 5.2 治理策略

- L1：允许先用后审（会话内可即时生效）。
- L2：需一次确认后执行，审计必须完整。
- L3：必须审批前置，不允许先用后审。

## 6. 关键数据模型增量（v2）

1. `ConversationAsset`
   - `assetType`、`title`、`payload`、`sourceTurnId`、`version`、`lineage`。
2. `ConversationAssetRef`
   - 表示当前回复引用了哪些历史资产。
3. `SkillDraftRuntimeGrant`
   - 记录低风险 Draft 在会话中的临时授权窗口。
4. `DeliveryChannelBinding`
   - 渠道类型、目标地址（群/用户）、认证配置引用。
5. `ConversationSchedulePolicy`
   - 对话任务计划、时区、静默时段、失败重试策略。

## 7. 对话交互规范（v2）

1. 默认自动执行：无阻塞确认。
2. 高风险动作采用内联确认卡片（不跳转页面）。
3. 每轮回复包含“可继续操作建议”（复用、交付、订阅、回测）。
4. 每条建议都可一句话触发，不要求用户填复杂表单。

## 8. 迭代计划（从当前实现继续）

### P1（1-2 周）对话内核与资产复用

1. 引入 `ConversationAsset` 与引用语义解析。
2. 将 plan/confirm 的用户可见流程折叠为系统内自动步骤。
3. 前端支持“引用上轮资产”快捷语义与可视化标签。

### P2（1-2 周）能力生长与风险门禁

1. Skill Draft 增加风险标签与 Runtime Grant。
2. 落地“低风险先用后审”可观测链路。
3. 增加审批拒绝后的自动回退与替代建议。

### P3（1-2 周）全渠道交付

1. 抽象 Delivery Adapter，统一发送协议。
2. 完成 DingTalk/WeCom/Feishu 通道接入。
3. 支持发送“原始文件”与“结构化摘要+附件”两种模式。

### P4（1 周）对话式定时任务增强

1. 自然语言转 cron + 时区 + 目标渠道。
2. 支持修改/暂停/恢复/一次性补跑。
3. 失败重试 + 超限告警 + 对话内可诊断提示。

## 9. 验收标准（v2 UAT）

1. 用户仅通过自然对话可连续完成一次复杂分析并得到可复用结果。
2. 用户可在 3 轮内完成“结果复用 -> 回测 -> 多渠道发送”。
3. 能力缺口出现时，系统能自动生成 Draft 并在低风险场景即时可用。
4. 用户可用一句话创建并修改定时任务。
5. 全链路具备审计：对话、技能调用、交付、调度、审批。

## 10. 当前代码基线与迁移策略

- 已有能力（会话、辩论、导出、邮件、订阅、回测、冲突、Skill Draft）作为 v2 的执行底座保留。
- v2 重点不在替换引擎，而在重构“对话内核 + 资产层 + 适配器层”产品体验。
- 迁移采用双轨：
  1. `copilot-v1` 保持稳定。
  2. 新增 `copilot-v2` 开关灰度验证并逐步替换。
