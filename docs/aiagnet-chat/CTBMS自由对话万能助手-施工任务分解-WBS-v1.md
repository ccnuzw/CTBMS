# CTBMS 自由对话万能助手施工任务分解（WBS）v1.0

## 0. 执行基线（2026-03-06）

1. 单一用户主入口：`智能体工作流 > 对话助手`（`/workflow/copilot`）。
2. 施工任务不得新增平行用户入口或把主流程迁出对话页。
3. 每个里程碑至少包含一条“提问 -> 得结果 -> 发送/定时”闭环验收任务。
4. 配套治理文档：
   - `docs/aiagnet-chat/CTBMS对话产品文档治理与落地执行计划-v1.md`
   - `docs/aiagnet-chat/CTBMS对话能力-功能菜单路由映射表-v1.md`

## 1. 施工目标

在保持现有系统能力可用的前提下，完成“普通用户无感 + 自由对话 + 复用优先 + 临时生成兜底”的完整改造。

---

## 2. 里程碑计划（4 周）

## M1（Week 1）体验层封板

1. 对话发送即时回显机制稳定上线。
2. 助手回复全部支持 `replyOptions`。
3. 普通用户视图术语清理完成。

## M2（Week 2）编排层增强

1. `createTurn` 接入 LLM 回复增强与兜底。
2. 选项生成语境化（结合意图、状态、结果）。
3. 对话追问策略“少问且准问”。

## M3（Week 3）能力层升级

1. 复用优先策略固化（Owner/Team/Public）。
2. 临时 workflow 生成链路上线（沙箱）。
3. 临时规则/参数 TTL 回收机制上线。

## M4（Week 4）治理与发布

1. 临时 skill 生成灰度上线。
2. 自动晋升/淘汰规则上线（首版）。
3. 全量门禁、灰度、回滚演练完成。

---

## 3. 任务分解（按模块）

## 3.1 Web（对话体验）

### W-1 输入与回显

1. 发送后本地用户气泡回显。
2. 助手处理中占位与失败重试。
3. 消息顺序一致性处理。

### W-2 回复与选项

1. 助手消息卡支持 `replyOptions`。
2. `SEND` 与 `OPEN_TAB` 行为统一。
3. 失败态与缺槽态默认选项兜底。

### W-3 普通用户视图

1. 隐藏专业模块（非管理员）。
2. 中文文案收敛与术语屏蔽。
3. 长内容摘要 + 展开体验优化。

## 3.2 API（会话编排）

### A-1 createTurn 协议升级

1. `replyOptions` 成为标准响应字段。
2. 回复结构固定化（结论/依据/建议）。
3. 风险分级确认策略接入。

### A-2 LLM 增强

1. LLM 输出 JSON 解析与兜底。
2. 开关控制（可快速关闭）。
3. 失败回退规则选项。

### A-3 路由与复用

1. 路由策略参数化（owner/public 优先级）。
2. 路由日志可查询。
3. 复用命中优先于新建。

## 3.3 Capability（能力池）

### C-1 临时 workflow

1. 生成入口。
2. 沙箱执行。
3. TTL 清理。

### C-2 临时 skill

1. 生成入口。
2. 安全检查。
3. 发布前审查策略。

### C-3 临时规则参数

1. 会话级参数包生成。
2. 持久化条件判定。
3. 自动过期回收。

---

## 4. 验收清单（逐项打勾）

## 4.1 功能验收

1. [ ] 发送后 200ms 内回显用户消息。
2. [ ] 每条助手回复存在下一步动作。
3. [ ] 普通用户界面无内部术语。
4. [ ] 复用命中时不重复创建能力。
5. [ ] 无命中时可生成临时能力并产出结果。

## 4.2 质量验收

1. [ ] api type-check 通过。
2. [ ] web type-check 通过。
3. [ ] agent-suite e2e 通过。
4. [ ] UAT/OPS/灰度/门禁报告生成通过。

## 4.3 运营验收

1. [ ] 关键指标埋点可见。
2. [ ] 灰度与回滚预案可执行。
3. [ ] 用户反馈渠道与问题分级机制建立。

---

## 5. 风险与缓解

1. LLM 波动：启用严格 JSON 输出 + 回退模板。
2. 临时能力膨胀：TTL + 指纹去重 + 自动淘汰。
3. 用户误操作：高风险动作一次确认。
4. 成本上升：会话级预算阈值与降级策略。

---

## 6. 发布建议

1. 先发布“体验层 + 编排层”能力。
2. 临时能力生成按模块灰度（先 workflow 后 skill）。
3. 每次放量前更新 4 份报告并门禁判定。
