# CTBMS 对话主线 US-504 UI 验收脚本 v1.0

- 日期：2026-03-06
- 目标：为 `US-504` 提供可重复执行的 UI 验收口径与留档模板
- 对应 Story：`US-504`（结果四段结构 + 对话内导出/发送/定时入口）

---

## 1. 自动化结构检查（必跑）

1. 命令：`pnpm agent:us504:ui-check`
2. 通过标准：
   - 结果四段文案存在：`核心结论`、`证据依据`、`风险提示`、`建议操作`
   - 对话入口存在：`发送报告`、`定时发送`
   - 详情分段存在：`发送`、`定时`
3. 失败处理：
   - 任一检查失败，判定 `US-504` 未通过，禁止放量。

---

## 2. 手工回放脚本（必跑）

### 2.1 前置条件

1. 已启动前后端：`pnpm dev`
2. 准备测试账号并进入：`/workflow/copilot`
3. 确保可创建并执行会话（建议先跑一次 `agent-conversation` e2e）

### 2.2 回放步骤

1. 输入问题：`分析最近一周东北玉米价格并给出建议`。
2. 等待结果生成，确认结果区显示四段结构：
   - 核心结论
   - 证据依据
   - 风险提示
   - 建议操作
3. 在会话主卡片中确认可见入口：
   - 查看结果
   - 发送报告
   - 定时发送
4. 打开会话详情抽屉，确认分段：
   - 结果
   - 发送
   - 定时
5. 在“发送”页执行一次发送（可使用测试地址/测试目标）。
6. 在“定时”页创建一个任务，并执行一次暂停或恢复。

### 2.3 通过标准

1. 四段结构全部可见。
2. 发送与定时入口均在对话页内可达，不需跳转复杂配置页。
3. 发送动作返回成功或可执行补救提示。
4. 定时任务可创建且状态可变更（暂停/恢复）。
