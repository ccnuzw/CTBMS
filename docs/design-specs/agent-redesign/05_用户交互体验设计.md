## 1. 前端组件架构 (Component Architecture)

基于 **Ant Design Pro** 和 **Atomic Design** 原则。

### 1.1 组件树结构 (Component Tree)

```text

├── WizardLayout (Provide: WizardContext)
│   ├── StepNavigator (Steps Component)
│   └── StepContentArea
│       ├── Step1_PersonaSelection

│       │   │   └── PersonaCard (Props: { icon, capabilities, tags })
│       │   └── FilterBar (Tags: Finance, Legal, Create...)

│       └── Step4_Playground
│           ├── ChatInterface (ProChat)
│           └── DebugPanel (Collapsible Trace View)
```

### 1.2 状态管理 (State Management)

使用 **XState** 管理复杂的向导流程，防止状态不一致。

```typescript
// wizardMachine.ts
const wizardMachine = createMachine({
  id: 'agentWizard',
  initial: 'personaSelection',
  context: {
    selectedPersona: null,
    credentials: {},
    files: []
  },
  states: {
    personaSelection: {
      on: { SELECT: { target: 'credentialBinding', actions: 'assignPersona' } }
    },
    credentialBinding: {
      on: { 
        SUBMIT: 'validatingCredentials',
        SKIP: 'knowledgeUpload' // If no auth needed
      }
    },
    validatingCredentials: {
      invoke: {
        src: 'validateApiKeys',
        onDone: 'knowledgeUpload',
        onError: 'credentialBinding' // Show error
      }
    },
    knowledgeUpload: {
      on: { NEXT: 'playground' }
    },
    playground: {
      type: 'final'
    }
  }
});
```

## 2. API 交互与实时反馈 (Data Fetching Strategy)

### 2.1 知识库处理进度 (SSE)
由于文件解析可能耗时，不能用简单的 HTTP Request/Response。
*   **Protocol**: Server-Sent Events (SSE)
*   **Endpoint**: `GET /api/knowledge/progress?batchId=xyz`
*   **Event Stream**:
    ```text
    event: progress
    data: {"fileId": "f1", "status": "parsing", "percentage": 30}
    
    event: progress
    data: {"fileId": "f1", "status": "indexing", "percentage": 80}
    
    event: complete
    data: {"fileId": "f1", "chunks": 120}
    ```

### 2.2 调试面板数据流
在 `Step 4` 沙箱中，Agent 的回复不再是单一字符串，而是结构化 Object。
*   **Response Schema**:
    ```typescript
    interface ChatResponse {
      content: string; // The actual reply
      traceId: string;
      steps: {
        type: 'tool_call' | 'retrieval' | 'reasoning';
        name: string; // e.g. "Binance API"
        input: any;
        output: any;
        latency: number;
        status: 'success' | 'error';
      }[];
    }
    ```
*   **UI Action**: 接收到 Response 后，前端渲染聊天气泡，同时更新侧边栏的 `Timeline` 组件。

## 3. 错误处理与用户引导 (Error UX)

### 3.1 鉴权失败
*   **场景**: 用户输入的 API Key 无效。
*   **UI**: 输入框变红，下方显示 "验证失败: 401 Unauthorized"。
*   **Action**: 提供 "如何获取 Key?" 的外链帮助文档。

### 3.2 知识库解析失败
*   **场景**: 上传了加密的 PDF。
*   **UI**: 进度条变红，显示 "解析失败"。
*   **Action**: 提示 "文件受密码保护，请移除密码后重试"。

## 4. 测试验收用例 (Test Cases)

### TC-01: 状态机流转
*   **Action**: 在 Step 1 选择 "需要 API Key" 的角色。
*   **Expect**: 点击 "下一步" 进入 Step 2 (Credential)。
*   **Action**: 在 Step 1 选择 "无需 API Key" 的角色。
*   **Expect**: 点击 "下一步" 跳过 Step 2，直接进入 Step 3 (Knowledge)。

### TC-02: 实时进度条
*   **Action**: 上传 50MB PDF。
*   **Expect**: 进度条从 0% -> 100% 平滑滚动，且中间不会出现 "HTTP Timeout" (因为是 SSE)。
