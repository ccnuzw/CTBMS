## 1. 智能切片器设计 (Smart Chunking Design)

### 1.1 核心接口定义 (Core Interface)

```python
from abc import ABC, abstractmethod
from typing import List, Dict

class DocumentChunk:
    content: str
    metadata: Dict[str, any]  # {source, page_num, section_title}
    embedding: List[float] = None
    parent_id: str = None     # For parent-child retrieval

class BaseParser(ABC):
    @abstractmethod
    def parse(self, file_path: str) -> List[DocumentChunk]:
        """
        Input: File Path
        Output: List of Chunks (Text + Metadata)
        """
        pass

# 策略模式：根据 mime-type 自动派发
class ParserFactory:
    @staticmethod
    def get_parser(mime_type: str) -> BaseParser:
        if mime_type == "application/pdf":
            return LayoutPDFParser() # 使用 Microsoft LayoutLM
        elif mime_type == "text/markdown":
            return MarkdownHeaderParser()
        else:
            return RecursiveTokenParser(chunk_size=500, overlap=50)
```

### 1.2 表格处理算法 (Table Recovery)
对于 PDF 中的表格，标准算法流程：
1.  **Detection**: 使用 `YOLOv8` 或 `PaddleOCR` 识别表格 Bounding Box。
2.  **Extraction**: 提取单元格文本，重构为 HTML `<table>` 结构。
3.  **Enrichment**: 将表头 (Header) 添加到每个单元格的语义中（例如：`2023 Q1` -> `Revenue: 2023 Q1`）。

## 2. 混合检索与 RRF 实现 (Hybrid Retrieval Implementation)

```json
      "vector": { "type": "dense_vector", "dims": 1536 },        // 向量检索
      "timestamp": { "type": "date" },
      "metadata": { "type": "object" }
    }
  }
}
```

### 2.2 RRF 融合公式 (Reciprocal Rank Fusion)
$$ Score(d) = \sum_{r \in R} \frac{1}{k + rank(r, d)} $$
*   `k`: 常数，默认 60。
*   `rank(r, d)`: 文档 `d` 在列表 `r` 中的排名。

**代码实现伪代码**:
```python
def reciprocal_rank_fusion(vector_results, keyword_results, k=60):
    scores = {}
    for rank, doc in enumerate(vector_results):
        scores[doc.id] = scores.get(doc.id, 0) + 1 / (k + rank)
    for rank, doc in enumerate(keyword_results):
        scores[doc.id] = scores.get(doc.id, 0) + 1 / (k + rank)
    
    return sorted(scores.items(), key=lambda x: x[1], reverse=True)
```

## 3. 任务队列设计 (Task Queue via BullMQ)

由于 Parse 和 Embedding 是 CPU/GPU 密集型操作，必须异步化。

### 3.1 队列定义
*   **Queue Name**: `knowledge-processing`
*   **Job Structure**:
    ```json
    {
      "jobId": "uuid",
      "fileId": "file_123",
      "pipeline": ["parse", "chunk", "embed", "index"],
      "config": { "chunkSize": 500 }
    }
    ```
*   **Concurrency**: 设置为 `Machine_Core_Count - 1`，避免阻塞 HTTP 服务。
