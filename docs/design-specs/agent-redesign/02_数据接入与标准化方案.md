## 1. 连接器作为代码 (Connector as Code)

连接器定义不再是简单的配置，而是遵循严格 Schema 的 YAML 文件，运行时动态加载。

### 1.1 Connector Schema Definition (Zod/TypeScript)

```typescript
// Type Definition for Connector Manifest
interface ConnectorManifest {
  meta: {
    id: string;          // e.g. "binance-ticker"
    version: string;     // e.g. "1.0.0"
    name: string;        // Human readable name
    category: 'FINANCE' | 'NEWS' | 'UTILITY';
  };
  
  auth: {
    type: 'NONE' | 'API_KEY' | 'OAUTH2';
    config?: {
      header_name?: string; // e.g. "X-MBX-APIKEY"
      token_prefix?: string; // e.g. "Bearer "
    };
  };

  endpoints: {
    id: string;
    method: 'GET' | 'POST';
    url: string;
    params?: Array<{
      name: string;
      required: boolean;
      in: 'QUERY' | 'BODY' | 'PATH';
    }>;
  }[];

  transform: ETLProcess[]; // Post-processing pipeline
}
```

### 1.2 鉴权实现策略 (Auth Implementation)

*   **API Key 注入**:
    *   Storage: 使用 AES-256 加密存储于 `connector_secrets` 表。
    *   Runtime: `HttpClient` 拦截器读取配置，自动注入 Header。
    
*   **OAuth2 Token 刷新**:
    *   利用 `BullMQ` 创建定期任务 `RefreshTokenJob`。
    *   逻辑：检查 `expires_at < Now + 5min` -> 调用 Refresh Endpoint -> 更新 DB。

## 2. ETL 算子与数据清洗 (ETL Operators)

为了让数据适应 LLM 上下文，我们预置以下原子操作符 (Operators)。

### 2.1 算子列表

| 算子ID | 输入示例 | 参数配置 | 输出示例 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| `json_pick` | `{"data": {"price": 100}}` | `path: "data.price"` | `100` | 提取核心字段 |
| `list_slice` | `[1, 2, ... 100]` | `start: 0, end: 5` | `[1, 2, 3, 4, 5]` | 截断过长列表 |
| `text_truncate`| `"Very long text..."` | `max_chars: 100` | `"Very long te..."` | 节省 Token |
| `unit_convert` | `1680000000` (ms) | `from: ms, to: iso8601` | `"2023-01-01..."` | 时间/单位转换 |
| `tmpl_render` | `{"price": 100}` | `tmpl: "当前价格为 {{price}}"` | `"当前价格为 100"` |以此结果做最终输出|

### 2.2 管道处理流 (Pipeline Execution)

```typescript
async function runPipeline(data: any, pipeline: ETLProcess[]) {
  let current = data;
  for (const step of pipeline) {
    switch (step.op) {
      case 'json_pick':
        current = jsonpath.query(current, step.args.path);
        break;
      case 'list_slice':
        current = current.slice(step.args.start, step.args.end);
        break;
      // ... handle other ops
    }
  }
  return current;
}
```

## 3. 缓存与限流设计 (Resilience)

### 3.1 Redis Key Design
*   **API Response Cache**:
    *   Key: `conn:cache:{connector_id}:{endpoint_hash}:{params_hash}`
    *   TTL: 动态配置 (e.g. 股价 10s, 财报 24h)
    
*   **Rate Limiter**:
    *   Key: `conn:limit:{user_id}:{connector_id}`
    *   Algorithm: Token Bucket (令牌桶) via `redis-cell` 或 Lua Script。

### 3.2 错误降级 (Fallback)
当 API 调用失败 (Status != 2xx) 时：
1.  **Retry**: 指数退避 (Exponential Backoff)，最多重试 3 次。
2.  **Stale Cache**: 如果 Redis 中有过期但在容忍范围内 (Stale-While-Revalidate) 的旧数据，优先返回旧数据并报警，而不是抛异常。
